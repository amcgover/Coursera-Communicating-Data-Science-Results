This notebook looks at the resolution rate of crimes with respect to:

  • Time of day
  • District
  • Type of crime

Resolved crimes are defined as those whose value for the 'Resolution' field is not 'NONE'. Resolution rate is defined as the ratio of resolved crimes to the total number of crimes.

First we load the libraries we'll need.

In [1]:
import pandas as pd
import matplotlib.pyplot as plt

Then we load the data. We remove 'non-criminal' and 'suicide' since these are not crimes against an individual.

In [41]:
sfCrime = pd.read_csv('sanfrancisco_incidents_summer_2014.csv')
# Remove 'NON-CRIMINAL' and 'SUICIDE' as these are not crimes against an individual
sfCrime = sfCrime[~sfCrime['Category'].isin(['NON-CRIMINAL','SUICIDE'])]

First we find the time of day with the lowest resolution rate. We do this by plotting an hourly time series of resolution rates. The first step is to convert the dataframe into the form we need it.

In [44]:
# Create a copy of the original data frame
forGraphing = sfCrime.copy()
# Convert the 'Time' column to a Python datetime object. We only keep the hour, not the minute.
forGraphing.Time = pd.to_datetime(forGraphing.Time, format="%H:%M").dt.hour

# Convert the 'Resolution' column to a number, where 0 means the value of the 'Resolution' field was NONE and 1 means the value
# of the 'Resolution' column was something other than 'NONE'
# We do this so that we can easily calculate the number of resolved crimes by summing the values in the 'Resolution' column
forGraphing.loc[forGraphing['Resolution'] != 'NONE', 'Resolution'] = 1
forGraphing.loc[forGraphing['Resolution'] == 'NONE', 'Resolution'] = 0
forGraphing['Resolution'] = pd.to_numeric(forGraphing['Resolution'])

forGraphing.Resolution = pd.to_numeric(forGraphing.Resolution)

# Calculate the total number of incidences for each hour
countedbyDate = forGraphing.groupby('Time',as_index=False).count()[['Time', 'IncidntNum']]

# Calculate the total number of resolutions for each hour
summedByDate = forGraphing.groupby('Time',as_index=False).sum()[['Time', 'Resolution']]

# Join the incidence counts with the number of resolutions for each hour
countsWithSums = countedbyDate.set_index('Time').join(summedByDate.set_index('Time'))

# Calculate the resolution rate by dividing the number of resolved crimes by the total number of incidences
countsWithSums['resolution_rate'] = (countsWithSums['Resolution'] / countsWithSums['IncidntNum']) * 100

We now plot resolution rate over time. We also plot incidence number as this will put resolution rate in context. For example, we will be able to see whether a particular hour has a high resolution rate simply because it has a low number of crimes.

In [45]:
countsWithSums['resolution_rate'].plot(figsize = (20,5), title = 'Resolution Rate', xticks=range(0,25))
plt.show()
countsWithSums['IncidntNum'].plot(figsize = (20,5), title = 'Number incidences', xticks=range(0,25))
plt.show()

We can see that period with the lowest resolution rate is 6-9pm. This appears to be in part related to the fact that there is a very high number of incidences in this time period. However, note that the number of incidences decreases from 6pm to 9pm but the resolution rate also decreases. So, even though the number of incidences is decreasing, the resolution rate is getting worse.
